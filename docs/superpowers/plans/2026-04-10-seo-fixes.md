## Task 1 — Fix CSP to allow Cloudflare Insights beacon

**File:** `public/_headers`

The current `script-src` directive in the `Content-Security-Policy` header does not include `https://static.cloudflareinsights.com`, causing the Cloudflare Web Analytics beacon to be blocked on every page load.

**Edit:** On the `Content-Security-Policy` line, change the `script-src` value from:
```
script-src 'self' 'unsafe-inline' 'unsafe-eval' https://www.googletagmanager.com https://www.google-analytics.com https://places.googleapis.com
```
to:
```
script-src 'self' 'unsafe-inline' 'unsafe-eval' https://www.googletagmanager.com https://www.google-analytics.com https://places.googleapis.com https://static.cloudflareinsights.com
```

The full updated `Content-Security-Policy` line in `_headers` should read:
```
  Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval' https://www.googletagmanager.com https://www.google-analytics.com https://places.googleapis.com https://static.cloudflareinsights.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; img-src 'self' data: https: blob:; font-src 'self' https://fonts.gstatic.com; connect-src 'self' https://www.google-analytics.com https://places.googleapis.com https://maps.googleapis.com https://static.cloudflareinsights.com; frame-src 'self' https://www.google.com https://maps.google.com
```

Note: Also add `https://static.cloudflareinsights.com` to `connect-src` since the beacon makes a network request to report analytics data.

**Verify:** After this change, opening the live site in a browser should show zero CSP-related console errors. The beacon cannot be tested locally — it only fires on Cloudflare Pages deployments.
